  • Explain the purpose of BOTS/SPIDERS.
  • BOTS, sometimes known as spiders, are automated pieces of code that crawl over the web and website, cataloging the pages they find for page ranking (web browser search results) or data collection.
  • How can you as a webmaster prevent your site from being crawled.
  • Adding a mandatory login page to access the website which prevents bots from getting in if they don't know the password, or using a robots.txt to ask the crawlers to kindly stop.
  • List 3 ways that you can improve the ranking of a web site.
    • Only include relevant and correct information on the site.
    • Making a nicely layed-out page that increases user retention.
    • Frequently updating the page to show recent activity.
  • List 3 ways that you can cause a web site to lose ranking.
    • Creating a giant network of pages that all link to each other to increase inlink/outlink counts, which when find out will get you IP banned.
    • Making a crappy website that is gross to navigate and drops user retention rates.
    • Onky spamming relevant keywords in the site description and body.

The Cloud The cloud refers to servers that are accessed over the Internet, and the software and databases that run on those servers.
Private Cloud A private cloud is a cloud computing environment dedicated to a single organization.
Public Cloud A public cloud is an IT model where public cloud service providers make computing services—including compute and storage, develop-and-deploy environments, and applications—available on-demand to organizations and individuals over the public internet.
Hybrid Cloud A computing environment that combines an on-premises datacenter (also called a private cloud) with a public cloud, allowing data and applications to be shared between them.
Thin Client A thin client is a computer that uses resources inside a central server as opposed to a hard drive.
Clustering A cluster is a group of inter-connected computers or hosts that work together to support applications and middleware (e.g. databases). In a cluster, each computer is referred to as a “node”. Cluserting is the act of creating these networks.
SaaS Service as a Software: A method of software delivery and licensing in which software is accessed online via a subscription, rather than bought and installed on individual computers.
PaaS Platform as a Servicea: A complete development and deployment environment in the cloud, with resources that enable one to deliver everything from simple cloud-based apps to sophisticated, cloud-enabled enterprise applications.
IaaS Infrastructure as a Service: a type of cloud computing service that offers essential compute, storage, and networking resources on demand, on a pay-as-you-go basis.
Elasticity Elastic computing is the ability to quickly expand or decrease computer processing, memory, and storage resources to meet changing demands without worrying about capacity planning and engineering for peak usage.
Scalability Scalability is the measure of a system's ability to increase or decrease in performance and cost in response to changes in application and system processing demands.
Data Security Data security is the process of safeguarding digital information throughout its entire life cycle to protect it from corruption, theft, or unauthorized access.
Privacy Privacy computing refers to a range of technologies and practices that prioritise the protection of individuals' privacy and data while using digital services and technologies. Essentially, it focuses on sharing the value of data without sharing (or risking the privacy and security of) the underlying data itself.
Download Speed The speed at which files are accessed, processed, and stored on a computer/network from the web/cloud
Upload Speed The speed at which files are accessed, processed, and sent to the web/cloud from a computer/network
Cluster A cluster is a group of servers and other resources that act like a single system and enable high availability, load balancing and parallel processing.
Client Server Architecture A computing model in which the server hosts, delivers and manages most of the resources and services to be consumed by the client.

  • 2. Describe how the web is constantly evolving (1.0, 2.0, 3.0)
  • 1.0: Web of Content, read-only 2.0: Web of the Users, read-write (JS, blogs) 3.0: Semantic Web, read-write-execute, prevalence of tools
  • 3. Identify the characteristics of the following:
    • - HTTP: Hypertext Transfer Protocol
    • A protocol used to transfer data across the internet
    • - HTTPS: Hypertext Transfer Protocol Secure
    • A protocol used to transfer data across the internet securely using the SSL encryption layer
    • - HTML: Hypertext Mark-up Language
    • Coding language used to create the bones of web pages using an XML-esque syntax
    • - URL: Uniform Resource Locator
    • A string that links to an IP, used for locating web pages
    • - XML: Extensible Mark-up Language
    • Used solely for transferring data, readable by many different languages
    • - XLST: Extensible Stylesheet Language Tranformations
    • Styling language for XML
    • - JavaScript
    • Coding language frequently used to give front-end functionality to websites. Also has backend uses with NodeJS
    • - CSS: Cascading Style Sheet
    • File with numerous lists attributing styles to different elements in a website's HTML; used for decoration basically.
    • - URI: Uniform Resource Identifier
    • The string after the domain name in a URL which specifies which resource to retrieve from the domain.
    • 4. Describe the purpose of a URL
    • A URL exists so that users on the internet do not need to know miscellaneous IP addresses of every website they want. Instead, they can remember a more human-readable string of characters, usually associated with the site in some way, and have DNS servers memorize the IPs for them.
    • 5. Describe how a domain name server functions
    • When a user requests to go to a URL, their HTTP request is sent to a DNS server to retrieve the IP attached to that URL. If the DNS server does not have the URL, it asks other DNS servers until one of them has the IP, and if they do (if not an error is sent back to the server and then the user) the server saves the URL-IP association. If the DNS server did have the IP or retrieved it from other servers, it then sends that IP back, and the user can then visit the website (assuming there is no error in finding the IP).
    • 6. Identify the characteristics of:
      • - IP - (Internet Protocol)
      • A protocol that uses various numbers to give addresses to computers on the internet. The numbers function just like house numbers; if another computer knows it, they know where to visit when navigating the internet.
      • - TCP – (Transmission Control Protocol)
      • A protocol used to send files across the internet with error checking. If any packets are found to be missing during the reconstruction after a FTP request (e.g. packet 6/1000 is missing), more requests and responses will be made to get those specific packets back.
      • - FTP - (File Transfer Protocol)
      • FTP sends files between computers with an internet connection. Breaks files down into packets, or small bits of data, and adds the sender IP, receiver IP, and packet number (like a count used for reconstruction of the file) to each. These packets are then sent to the receiver and reassembled. Uses TCP for error checking. Then the requester can respond with their own files or information in the same way.
